Semana 1: Fundamentos de bases de datos

En esta primera semana sentamos las bases: qué es una base de datos, qué hace un sistema gestor (SGBD) y por qué son críticos en cualquier sistema de información profesional.

1. Conceptos fundamentales

Antes de diseñar modelos y escribir consultas SQL, necesitamos un lenguaje común sobre datos, información, bases de datos y SGBD.

Dato: representación simbólica (número, texto, fecha, código) de un hecho, sin contexto.
Información: dato procesado y contextualizado que responde a una pregunta y sirve para tomar decisiones.

Una base de datos es un conjunto organizado de datos relacionados, almacenados de forma que minimizan la redundancia, preservan la integridad y facilitan el acceso eficiente.

Un Sistema Gestor de Bases de Datos (SGBD) es el software especializado que permite definir, construir, manipular y administrar una base de datos (ej.: PostgreSQL, MySQL, SQL Server, MongoDB).

1.2. Tipos de usuarios de base de datos

Administrador de BD (DBA)

  • Rol: Configura y mantiene el SGBD.
  • Tareas: Crear esquemas, gestionar usuarios, respaldos, rendimiento.
  • Enfoque: Disponibilidad, seguridad e integridad de los datos.

Desarrolladores y usuarios finales

  • Desarrollador: diseña el modelo de datos y consultas, integra la BD en la aplicación.
  • Usuario final: interactúa con los datos a través de formularios, reportes y aplicaciones.
  • Enfoque: Funcionalidad, usabilidad y soporte al negocio.

2. Archivos planos vs SGBD

Muchos sistemas pequeños comienzan con hojas de cálculo o archivos de texto. A medida que el sistema crece, aparecen problemas que un SGBD resuelve de forma nativa.

AspectoArchivos planos (Excel, CSV)SGBD (PostgreSQL, MySQL, etc.)
RedundanciaAlta: el mismo dato se copia en muchas hojas/archivos.Baja: diseño normalizado y relaciones entre tablas.
IntegridadDepende de la disciplina del usuario.Restricciones, claves primarias y foráneas, reglas.
ConcurrenciaDifícil: riesgo de sobrescribir archivos compartidos.Manejada por el motor (bloqueos, transacciones).
EscalabilidadPoco adecuada para grandes volúmenes de datos.Diseñados para manejar millones de registros.
ConsultasLimitadas, poco flexibles.Lenguaje SQL potente y estandarizado.

Caso de estudio

Del Excel al SGBD en un sistema académico

1. El escenario

Una pequeña institución educativa lleva el control de estudiantes, asignaturas y notas únicamente en hojas de cálculo. Después de algunos semestres aparecen los siguientes síntomas:

  1. Listas de estudiantes duplicadas y con nombres escritos de forma distinta.
  2. Notas inconsistentes entre el acta del docente y el reporte enviado a coordinación.
  3. Dificultad para saber el historial completo de un estudiante en todas las asignaturas.
  4. Riesgo alto de pérdida de información al sobrescribir archivos.
